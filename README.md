# Site Educacional: Café Arábica

**Aluno:** Eduardo Henrique Ramos Coloni  

## ☕ Sobre o Projeto
Este é um projeto front-end completo e responsivo dedicado a apresentar informações sobre o café arábica. O site foi desenvolvido com foco rigoroso em **Acessibilidade, Performance, Melhores Práticas e SEO**

## 🚀 Tecnologias Utilizadas
* **HTML5:** Estrutura 100% semântica (`<header>`, `<nav>`, `<main>`, `<section>`, `<article>`, `<footer>`).
* **CSS3:** Estilização customizada avançada, com suporte nativo a Tema Claro/Escuro (Dark Mode) respeitando regras de contraste WCAG.
* **JavaScript (Vanilla):** Manipulação da DOM para interatividade e validações avançadas sem uso de bibliotecas externas.
* **Bootstrap 5:** Sistema de grid para responsividade e componentes visuais (Cards, Navbar, Modal).
* **FontAwesome:** Ícones vetoriais.

## ✨ Principais Funcionalidades

### 1. Validação de Formulário Avançada (Tempo Real)
O formulário de inscrição não depende do botão "submit" para checar os dados. O JavaScript valida os campos instantaneamente enquanto o usuário digita:
* **Nome/Sobrenome:** Verifica quantidade mínima de caracteres.
* **Email:** Validação de formato (exigência de provedor após o `@`).
* **Telefone:** Máscara de DDD e limite automático de caracteres.
* **CPF:** Algoritmo matemático oficial que bloqueia CPFs falsos (ex: 111.111.111-11) e aplica máscara visual.
* **Senha Dinâmica:** Checklist visual que fica verde apenas quando a senha atinge 4 requisitos simultâneos (8+ caracteres, maiúscula, número e símbolo especial).

### 2. Experiência do Usuário (UX / UI)
* **Toggle de Senha (Olhinho):** Implementado tanto no formulário de inscrição quanto no modal de Login, permitindo visualizar e ocultar a senha digitada.
* **Scroll Suave:** Navegação por âncoras na navbar que desliza a tela suavemente até a seção desejada.
* **Feedbacks Visuais:** Bordas dos inputs ficam vermelhas (is-invalid) ou verdes (is-valid) de acordo com o preenchimento, melhorando a compreensão do usuário.

### 3. Otimização de Performance e Acessibilidade
* Imagens convertidas para o formato de nova geração (`.webp`), garantindo carregamento ultrarrápido (redução drástica no LCP).
* Uso de atributos `preload`, `fetchpriority="high"` e `loading="lazy"` para gerenciar a prioridade de download dos recursos.
* FontAwesome e estilos não-críticos carregados de forma assíncrona para não bloquear a renderização da página inicial (FCP).
* Contraste de cores perfeitamente ajustado tanto para a versão clara quanto para a versão escura do site.

## ⚙️ Como Executar o Projeto

Como este é um projeto estático (Front-end puro), não há necessidade de instalar dependências complexas ou configurar bancos de dados.

1. Extraia os arquivos do projeto.
2. Abra o arquivo `index.html` diretamente em qualquer navegador moderno (Chrome, Edge, Firefox).
3. *(Opcional)* Para uma experiência de desenvolvimento, abra a pasta no VS Code e utilize a extensão **Live Server**.
